The 5 "W's"

  • Who?
  • What?
  • Where?
  • When?
  • Why?
  • ... and How?

Who?

Developers can/should use them

Opsy people can help prepare them, share them

What?

Automated, Repeatable, Consistent development environments

Where?

You PC. Your MAC. You linux box. Maybe even your CI environment!

When?

Now, and since 2012, and earlier.

Why?

  • "It works on my machine"
  • New devs can spend 3 days configuring the environment
  • Polluted computer w/ 6 versions of java, 18 of ruby, 14 gemsets, 3 pythons, etc

Why? Continued

  • Independent environments (no pollution)
  • New devs up and running quickly (repeatability)
  • Everyone runs the same thing, ideally the same as production (consistency)

How?

  • Virtualisation/Isolation
    • VirtualBox
    • VMWare
    • KVM
    • Docker, Docker, Docker!
    • chroot jail, or whatever
  • Automation
    • Vagrant
    • Puppet/Chef/Ansible/Salt/or whatever

psst, I don't really care which! But I can help you best w/ Vagrant.

Vagrantfile

$ vagrant init
Vagrant.configure(2) do |config|
  config.vm.box = "ubuntu/trusty64" # base machine
  config.vm.network( "forwarded_port",
	guest: 80, host: 3000
  )

  config.vm.provision( "shell",
	path: "build/vagrant-box/install-rvm.sh", privileged: false
  )

  config.vm.provision( "shell",
	path: "build/vagrant-box/install-ruby-21.sh", privileged: false
  )

  config.vm.provision( "shell",
	inline: "gem install bundler", privileged: false
  )

  config.vm.provision( "shell",
	inline: "bundle install", privileged: false
  )
end
